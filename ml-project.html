4. Explanation of choosen hyperparameters¶

How did we determine which hyperparameters would yield the model with the highest possible Macro F1 and the fewest critical errors? We used the function below, which relies on cross-validation, splitting the training set into cv=5 folds. This allowed us to select the combination of parameters from a given range that maximized model performance. After testing, we set max_depth to 7, as this configuration produced better results in terms of the confusion matrix, reduced overfitting, and a higher Macro F1-score, while minimizing critical errors.

In [270]:
from sklearn.model_selection import GridSearchCV
from sklearn.tree import DecisionTreeClassifier
dt = DecisionTreeClassifier(random_state=42)

param_grid_dt = {
    'max_depth': [3, 5, 6, 7, 10],
    'min_samples_split': [5, 10, 20],
    'min_samples_leaf': [ 5, 10, 20], 
    
}
grid_search_dt = GridSearchCV(
    estimator=dt,
    param_grid=param_grid_dt,
    cv=5,              
    scoring='f1_macro', 
    n_jobs=-1           
)

grid_search_dt.fit(features_train, outcome_train)
print("Best parameters:", grid_search_dt.best_params_)
print("Best F1 macro:", grid_search_dt.best_score_)

#n_jobs is a parameter that tells how many processor cores should be used to perform calculations in parallel.
# setting this to -1 each combination of hyperparameters or folds in cross-validation is calculated in parallel, instead of sequentially.
# therefore GridSearchCV works muchfaster

5. Conclusions¶

The model performs well for the extreme classes, Green and Red, but struggles with the intermediate classes, Yellow and Orange, which are frequently misclassified. This results in a moderate macro F1 score of 0.506. The low performance on Yellow and Orange suggests that a linear decision boundary is insufficient to separate these classes effectively. To improve overall performance, a nonlinear model, such as Random Forest, could better capture the complex relationships between features and classes.

Due to the lack of linear separability and the presence of overlapping classes, the LinearSVC optimizer required a high number of iterations to converge. The maximum number of iterations was increased to 10,000,000 to eliminate convergence warnings and ensure numerical stability of the solution.

General conclusions¶

  1. Random Forest (F1 macro = 0.8846) performed the best among all tested methods. It effectively captures complex, non-linear relationships between features, handles outliers well, and provides stable predictions due to its ensemble nature.

  2. Decision Tree (F1 macro = 0.8143) also performs well but is more prone to overfitting and may struggle with underrepresented classes.

  3. KNN (F1 macro = 0.78308) gives moderate results, performing better on dominant classes but struggling with rarer classes, especially if features are not perfectly scaled.

  4. Gaussian Naive Bayes (F1 macro = 0.6594) performs worse due to the assumption of feature independence, which likely does not hold in this dataset.

  5. SVM (F1 macro = 0.4971) performs the worst. Linear SVM cannot separate intermediate classes effectively, indicating the data is not linearly separable. A non-linear kernel (e.g., RBF) or a different model may improve results.

Summary: Random Forest is the most suitable method for this dataset, balancing accuracy and stability, while other models are limited by linearity assumptions, sensitivity to feature scaling, or overly simplistic modeling of feature relationships.
